Feedback

Technical Description

Firstly, declares the document into HTML5. Then add document's language into English and meta data. In the header tag finally add the styling links.

The body contains the main content. header contains the site logo, navigation bar, and menu buttons. In the navigation bar it includes a list ul tag with li items for navigation links.

Form Structure

In this html code there are mainly three Fieldsets. Legend provides a title for each Fieldset (Personal Details, Ocean Awareness and Submit).

Label is used for form each element description and input fields provide the types like text, email, tel, date-time inputs. Textarea allows multi- line text input. Select creates dropdown with option element for choices. Datalist provides predefined suggestions for an input field. Input type radio and checkbox enable single or multi selections. Pattern, title, and required attributes enforce input validation. Aria label in the file upload input improves accessibility. For rate about the marine life I have used five stars.

Css

In the global css it includes the common css stylings such as navigation bar styling, footer styling and background colour etc. For the feedback form fieldset background color is whitesmoke for better readability. Border radius has rounded the edge. Then width, margin left, margin right centers the form.

In the css code there are plenty of classes and ids. For each class and ids, I have used different stylings for better user experience. The form includes different classes and ids for dropdowns, textareas, star-based rating systems, reset and submit buttons valid and error messages etc. For the star-based rating system it includes interactive hover and click effects. Error messages and validation styling are incorporated for user-friendly input handling. Buttons have a distinct color, and a word counter is positioned within textareas.

Accessibility

In the feedback form accessibility has been addressed through multiple features. Fieldset, legend , label elements improves the screen reader compatibility. Input fields includes aria-label attributes and descriptive placeholders for text for better clarity. For the validation purpose I have used title attribute and error messages to help users to correct mistakes.

Team Page

Technical Description

HTML

Firstly declares the document into HTML5. Then add document's language into English. In the header tag finally add the styling links.

The body contains the main content. header contains the site logo, navigation bar, and menu buttons. In the navigation bar it includes a list ul tag wiith li items for navigation links.

Section class team containing the team details. It started with a main heading h1 and a subheading h3 tag. Each team member’s details are display in the div class box. Inside that there is img tags and hidden details about team members (name, role and design and implanted pages). The onclick and onkeydown events trigger JavaScript functions to toggle the visibility of these details, allowing users to view more information interactively.

In the html code I have used heading tags to display the hierarchy. The tabindex="0" attribute allows the images to be focusable by keyboard users, and the aria-expanded="false" attribute provides screen readers with information about the visibility state of the team member details.

The footer links team page to student3 page editor page.

Css

In the global css it includes the common css stylings such as navigation bar styling, footer styling and background colour etc.

The .team class positions the entire section centrally within the viewport using flex, aligning items both horizontally and vertically. The .team content class displays a grid layout for the team members. Its layout with grid-template-columns: repeat(auto-fit, minmax(150px, 1fr)), ensuring responsive columns that adjust based on screen size. .team-comtent imag class display the image with rounded corners using border radius. The .box class defines the background colour for the box with rounded corners and smooth transition effects. The .hidden-details class ensures to hide the additional details by default and only displayed when toggled with the .show class.

Additionally, css code includes hover effects, different font sizes and colours to specific areas , text styles etc.

Accessibility

In the team page there are several accessibility features. Firstly, inside the img tag there is an attribute called alt. This alt tag includes alternative information for an image. (Student 1, Student 2, Student 3, and Student 4) That is a text description of an image that is read aloud by screen readers. Additionally, each team member’s image has a “tabinex=0” attribute. This allowing user to navigate through the images using the keyboard. To further enhance accessibility, the aria-expanded="false" attribute is used on the images, indicating whether the associated details are visible or hidden, and it is dynamically updated via JavaScript when the details are toggled. Pressing Enter or space key keyboard users can toggle the details because I have used onkeydown event attribute.

Finally, the whole HTML tags have a logical flow.

Content Page

Technical Description

HTML

Firstly, declares the document into HTML5. Then add document's language into English. In the header tag finally add the styling links.

The content page includes standard layout including header , content sections and footer. Header contains the site logo, navigation bar, and menu buttons. In the navigation bar it includes a list ul tag with li items for navigation links. For the content sections I have used different headings according to the hierarchical order. In the content page sections are clearly defined using div tags with corresponding classes. The images are added using the img tag, enclosed within figure tags for better accessibility and semantic grouping with captions provided through figcaption.

The content page also includes internal linking. Anchor tag has been used for the internal navigation. This is supported by the id attributes on the headers like id= BiologicalClassification allowing users to go to specific sections of the webpage.

Finally, footer element links the content page to student3 page editor.

CSS

In the global css it includes the common css stylings such as navigation bar styling, footer styling and background colour etc.

.container class is wrapping the main content and providing styling such as line height, width, margin, and box shadow. The padding and margin elements make the webpage more readable. For the navigation navlist id is used to make a horizontal navigation bar. It includes hover to change the colour when click a link. images and .marine, .sea classes are responsible for style the images. .top class is used to style the back to top button.

Accessibility

In the team page there are several accessibility features. Firstly, inside the img tag there is an attribute called alt. This alt tag includes alternative information for an image. (For the content page images it includes marine creature’s name.) Figcaption element also a accessibility feature. Navigation links can be easily identified with anchor tag and provide smooth internal navigation via id references.

Challenges and Lessons Learned

I encountered difficulties when creating this webpage called “Life below water’’. Main challenge that I have faced is handling various coding styles and guaranteeing seamless collaboration during the webpage's development. When adding styling to corresponding webpages we had many issues. We fixed this issue by creating a global.css file. Key lesson learned was the importance of clear communication and setting up proper version control early in the project to avoid confusion and streamline the development process.

Compliance

By adding alt text for photos to provide accessibility, the webpage I made conforms with Janet standards. This guarantees that people who are visually impaired can comprehend the images' information. According to the requirements, alt text was the focus, although other compliance factors like security and privacy were taken into account generally.
